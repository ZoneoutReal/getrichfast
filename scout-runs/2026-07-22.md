# Scout run — 2026-07-22

First real run of the niche scout, against live web research (not the example
data). Recorded here as the factory's decision log. **Integrity note:** the
scout deliberately does not scrape the Chrome Web Store or stats aggregators
(both 403 automated fetches, and scraping them would break niche gate #4), so
exact per-extension install counts below are *category-level estimates from
public search results* and are flagged where soft. Precise counts get confirmed
from the store listing / dashboard before a build starts.

## Headline finding

The "pay-once + 100% local" wedge that carried products #1–#7 is **getting
commoditized in the easy categories.** For PDF and image conversion, the big
paid incumbents are *web tools that upload your files* (a real privacy wedge),
but the Chrome extension shelf is already **crowded with free, privacy-first
local clones**. That means a new pay-once extension there competes against free
— a race to the bottom, not a wedge.

This is the scout earning its keep: it steers away from headline demand that
has no durable monetization, toward the PLAYBOOK's own **next-wave thesis** —
differentiate on a *capability* the free clones lack (in-browser ML: OCR,
transcription, voice-typing), where clones are far fewer and the privacy story
is load-bearing.

## Evidence gathered

**PDF tools**
- Smallpdf / iLovePDF: subscription (~$9–14/mo), **upload files to their
  servers**, free tier ~2 tasks/day — a genuine privacy + price wedge.
- But the extension shelf already has many **free, no-upload, local** PDF tools
  ("Offline PDF Tools", "pdfed", "Local PDF Toolkit", "Silent Editor", etc.),
  and individual merge/split extensions are small (e.g. "Super PDF Split Merge"
  ≈ 1,000 users). Some free ones carry malware warnings in reviews.
- Read: demand real, **monetization contested** by free local clones. Build
  only with a capability moat (true redaction, OCR) the free tools lack.

**Image / HEIC conversion**
- Same shape: many **free "100% local, no upload"** HEIC→JPG/WebP converters
  already on the store. Commodity capability, weak pay-once wedge.

**Clipboard managers** (relevant to shipped product #10, ClipStack)
- "Clipboard History Pro" ≈ **130k+ users**, free with emerging premium.
  Confirms the category demand ClipStack targets, and that the incumbent is
  free/freemium — so ClipStack's Pro must win on features the free leader
  lacks (pinned collections, image clips, local-only guarantee).

## Scout verdict

- **PDF / HEIC as plain pay-once extensions:** qualify on demand but flagged
  `monetizationRisk` by the new free-clone-saturation signal. Not recommended
  as-is.
- **Recommended direction:** prioritize the **capability-moat wave** already in
  PLAYBOOK.md → "Next wave" (local OCR / screenshot-to-text, local
  transcription, voice-typing into any field). Harder to build → far fewer
  me-too clones → the privacy claim becomes the product, not just the price.

## Tooling improvement shipped from this run

Using the scout for real exposed a gap: gate #2 only checked whether a *paid*
incumbent exists, so a niche saturated with *free* clones still scored as a
clean wedge. Added a **free-clone-saturation penalty**: competitors priced free
at ≥10k installs now discount the wedge score and raise a `monetizationRisk`
flag surfaced in the build issue. (Tested; see `scripts/tests/scout.test.mjs`.)
