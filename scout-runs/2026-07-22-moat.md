# Scout run — 2026-07-22 (capability-moat candidates → product #11)

Second real scout run, this time evaluating the PLAYBOOK "next wave" candidates
head-to-head to pick product #11. Same integrity note as the first run: no store
scraping, install counts are order-of-magnitude estimates from public search
results, confirmed before a build starts.

## Result (scored by `node scripts/niche-scout.mjs`)

| Rank | Candidate | Qualified | Score | Monetization risk | Note |
|---|---|---|--:|---|---|
| 1 | **Private Screen Recorder** (local, pay-once) | ✅ | **70** | low | incumbents are *paid cloud*, not free clones |
| 2 | Local QR generator + reader | ✅ | 68 | **high** | popular extensions are free |
| 3 | Local EXIF / metadata remover | ✅ | 45 | **high** | shelf is mostly free |
| 4 | Local OCR / screenshot-to-text | ❌ | — | high | **fails gate #2** — no paid wedge, all free |

## The finding

The "easy" capability plays are already **given away free** on the extension
shelf. OCR is the clearest case — Copyfish, Text Capture OCR, Image-to-Text are
all free and local, so there is no monetization wedge at all (the scout
disqualified it on gate #2). EXIF removal and QR generation are the same story:
they qualify, but the free-clone-saturation signal flags both `high` risk
because the popular tools are free. Building a pay-once tool into any of those
is a race to the bottom — exactly the trap the scout exists to catch.

**Screen recording is the exception, and that is the whole point.** There the
incumbents are *paid cloud* products — Loom (subscription, uploads every
recording, free tier capped at 5 min / 25 videos), Screencastify (freemium,
cloud, watermark), Awesome Screenshot (freemium, cloud). None of them is a free
*local* tool. So "record to your own device, no upload, no account, pay once"
is a genuine, on-brand wedge that the free clones have **not** eroded — which is
why it is the only candidate that scores a `low` monetization risk.

## Recommendation: build product #11 = a local, private screen recorder

- **Wedge:** the privacy/local + pay-once angle against upload-everything,
  subscription cloud recorders. Fits the portfolio's identity exactly.
- **Pricing:** one-time $19 (recording tools command more than the $9–15 utils).
- **Free vs Pro (proposed, no dark patterns — free core stays genuinely useful):**
  - *Free:* record screen / window / tab + microphone, live timer, 100% local,
    save as `.webm`. No watermark, no length cap, no account.
  - *Pro ($19):* capture tab/system audio, pause & resume, countdown + click
    spotlight, region record, and quality/format options.

### Why this run stopped at "recommendation" rather than a built PR

A screen recorder is the one candidate with real architectural weight and a
support-risk the scout flagged `medium`, so it warrants a deliberate build, not
an overnight one:

- **Architecture:** MV3 can't hold a recording in the popup (it dies on blur).
  The correct pattern is `chrome.desktopCapture.chooseDesktopMedia(...)` →
  `streamId` → a **chrome.offscreen** document running
  `getUserMedia({video:{mandatory:{chromeMediaSource:'desktop', chromeMediaSourceId}}})`
  → `MediaRecorder` → `chrome.downloads`. Several moving parts.
- **Testability:** the record engine (MediaRecorder → webm) can be tested for
  real by feeding it a `canvas.captureStream()` in Playwright (no display
  permission needed) — but the `desktopCapture` picker flow can only be
  verified by hand. That manual capture check should be part of the build.
- **Product decision:** the Free/Pro line above is a founder call worth a look
  before it ships.

**Greenlight and I'll build it to factory standard** (offscreen + desktopCapture
architecture, canvas-stream engine tests, icons/screenshots/listing/zip,
`check-release` pass) on a fresh branch as a draft PR.
