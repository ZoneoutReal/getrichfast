Questions, answered.

Can FocusFence see my browsing?

No. Blocking happens inside Chrome\'s declarativeNetRequest engine — the browser evaluates the rules itself. FocusFence never observes requests, never reads page content, and has no servers to send anything to.

Why does it ask for access to all sites at install?

Chrome requires host access for an extension to redirect a blocked navigation to its local fence page (the one with the countdown) instead of a raw error screen. FocusFence runs no code on websites — the permission exists purely so the redirect target is allowed. The code is open on GitHub if you want to verify.

What happens when I hit a blocked site?

The navigation is stopped by the browser before any request leaves, and you land on a calm local page showing which site was fenced, your custom message (Pro), and the session countdown.

Can I get around strict mode?

Not from inside FocusFence — while a session runs with strict mode on, the list and switches are frozen and there\'s no "end early" button. (Chrome always lets you remove an extension entirely; strict mode is a commitment device, not a jail.)

Is Pro really one-time?

Yes. $15 once, all Pro features forever, every future update included. Payments are processed by Stripe via ExtensionPay; we never see your card details.
