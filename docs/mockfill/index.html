Questions, answered.

Does MockFill ever fill or submit forms on its own?

Never. It only acts when you press the shortcut, click the button, or use the context menu — and it never submits anything. Chrome's activeTab permission means it can't even see a page until you invoke it there.

Where does the fake data come from?

It's generated locally, on your machine, every time. There's no server, no dataset download, no account, and no analytics. Emails use reserved example domains and phone numbers use the 555-01xx block reserved for fiction, so test data can never reach a real person.

Are the Pro test credit cards real?

No — they're the official, publicly documented test numbers from payment processors (like 4242 4242 4242 4242). They're Luhn-valid so your validation accepts them, but they can never be charged.

Is Pro really one-time?

Yes. $15 once, all Pro features forever, every future update included. Payments are processed by Stripe via ExtensionPay; we never see your card details.

Does it work with React / Vue / Angular forms?

Yes. Values are set through native setters and followed by real input/change events, which is exactly what framework change-tracking listens for.
